# Markdown2Word

AIで作成したMarkdownテキストをWordファイル（.docx）に一発変換するツールです。

---

## 使い方

### 基本手順

1. `Markdown2Word.exe` を起動する
2. AIのチャット画面でMarkdownをコピーする
3. 「クリップボードから貼り付け」ボタンをクリック
4. テンプレートを選択する（後述）
5. 出力ファイル名を入力する
6. 「Wordに変換して開く」ボタンをクリック

### ファイルから読み込む場合

- 「ファイルから読み込み」ボタンで `.md` ファイルを選択
- または `.md` ファイルを `Markdown2Word.exe` にドラッグ&ドロップ

---

## テンプレートは必須ではありません

**テンプレートがなくても、そのまま使えます。**

テンプレートの有無で動作は変わりません。違いはデザインだけです。

| 状態 | 変換結果 | こんな人向け |
|------|---------|------------|
| テンプレートなし | Wordのデフォルトスタイルで変換される | とにかく今すぐ使いたい |
| テンプレートあり | 会社指定のデザインが自動適用される | フォーマットを統一したい |

まずはテンプレートなしで試してみて、「デザインも整えたい」と思ったときにテンプレートを作るのがおすすめです。

---

## テンプレート（スタイルファイル）の作り方

テンプレートとは「フォント・色・サイズなどのデザインのお手本」となるWordファイルです。
一度作っておくと、変換するたびに会社指定のフォーマットが自動適用されます。

### 手順

**① 普通にWordを開いて、スタイルを設定する**

Wordの「ホーム」タブ →「スタイル」から各スタイルを右クリック →「変更」で編集します。

| Wordのスタイル名 | 対応するMarkdown | 設定例 |
|-----------------|-----------------|--------|
| 見出し 1        | `# テキスト`    | 青色・太字・18pt |
| 見出し 2        | `## テキスト`   | 紺色・太字・14pt |
| 見出し 3        | `### テキスト`  | 黒・太字・12pt |
| 標準            | 本文テキスト    | 游明朝・10.5pt |
| 表のスタイル    | テーブル        | ヘッダー行に色付けなど |

> ※ スタイル名は**必ずこの名前**（見出し 1、見出し 2 など）のままにしてください。
> 名前を変えると変換後に反映されません。

**② ダミーの内容を書いておく（推奨）**

変換結果のイメージを確認しやすくなります。実際の文章は不要で、スタイルを当てたサンプルテキストだけで構いません。

例：
```
（見出し1スタイルを当てた行）サンプル見出し1
（見出し2スタイルを当てた行）サンプル見出し2
（標準スタイルを当てた行）本文テキストのサンプルです。
```

**③ 名前を付けて保存する**

- `style.docx` → デフォルトのテンプレートとして使用されます
- `templates/報告書.docx` → テンプレート選択に「報告書」として表示されます
- `templates/議事録.docx` → テンプレート選択に「議事録」として表示されます

---

## フォルダ構成

```
Markdown2Word.exe         ← 実行ファイル
style.docx                ← デフォルトテンプレート
templates/                ← 複数テンプレートを置くフォルダ
    報告書.docx
    議事録.docx
    提案書.docx
```

> `templates` フォルダは自分で作成してください。

## exeのビルド方法（開発者向け）

### 必要なもの

- Python 3.x
- Pandoc（https://pandoc.org/installing.html）
 　※pandoc-3.9.0.2-windows-x86_64.msiを選ぶ。

- 必要なPythonライブラリ

```
pip install pypandoc pyinstaller
```

### ビルドコマンド

`Markdown2Word.py` があるフォルダで以下を実行：

```
pyinstaller Markdown2Word.spec
```

`dist/Markdown2Word.exe` が生成されます。

> **注意：** `style.docx` や `templates` フォルダは exe と**同じフォルダ**に置いてください。
> exeの中には同梱されません（ビルド時に存在しなかった場合）。

---

## 注意事項

- **Pandocが必要です。** 配布先のPCにもPandocがインストールされている必要があります。
  インストーラー：https://pandoc.org/installing.html
- 画像は Markdown ファイルからの相対パス・絶対パス・URL が使えます。
- 出力先フォルダを指定しない場合は**デスクトップ**に保存されます。
